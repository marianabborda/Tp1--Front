Decisiones de Diseño

Estética

  • Estética Unificada: Se optó por una base estética común basada en un "Dark Mode" profundo, utilizando una paleta de colores violetas y azulados para transmitir una imagen moderna y tecnológica.

  • Tipografía de Contraste: Se decidió combinar la fuente "Playfair Display" para títulos (aportando elegancia y legibilidad) con "DM Sans" para el cuerpo de texto, priorizando la claridad en la lectura.

  • Estructura Híbrida: Se tomó la decisión estratégica de que cada integrante utilizara esta base estética común pero aportara su propia estructura de maquetación y selección de contenidos para sus páginas personales. Esto permite que el sitio refleje la personalidad y el enfoque técnico individual de cada miembro dentro de un marco visual profesional.

Responsive Design

Implementamos breakpoints específicos según los requerimientos:

  • 400px: Dispositivos móviles pequeños
  • 900px: Tablets y pantallas medianas
  • 1200px: Pantallas de escritorio grandes

Utilizamos CSS Grid y Flexbox para layouts adaptables.

Dificultades y Soluciones

Adaptabilidad del contenido individual

Problema: Al permitir que cada integrante gestione su propia estructura y contenidos de forma autónoma, surgieron inconsistencias en la extensión de los textos y los formatos de los elementos. Esto provocaba que, en ciertas resoluciones, el diseño se desbordara o quedaran espacios vacíos antiestéticos, comprometiendo la navegación en dispositivos móviles.


Solución: Se implementó un sistema de diseño flexible combinando CSS Grid para la estructura general de las tarjetas y Flexbox para la alineación interna de los elementos. Estas técnicas permitieron que los contenedores se ajusten automáticamente al volumen de contenido de cada miembro, asegurando que el sitio sea totalmente responsivo y mantenga la armonía visual sin importar la cantidad de información incluida.

Sincronización de scripts con la carga del DOM:

Problema: Al implementar funciones interactivas como el efecto de "scramble" en el título o el efecto magnético en las tecnologías, los scripts intentaban ejecutarse antes de que los elementos HTML estuvieran completamente renderizados en el navegador. Esto provocaba errores en la consola (como TypeError: cannot read property of null) porque JavaScript no encontraba los selectores de clase como .hero h1 al momento del inicio.


Solución: Se implementaron dos estrategias. Primero, se colocó la etiqueta al final del archivo HTML, justo antes del cierre del , asegurando que el contenido se cargue primero. Segundo, se incluyeron validaciones preventivas dentro de cada función, como if (!hero || !heading) return;, lo que permite que el código sea robusto.

Gestión del rendimiento en animaciones de alta frecuencia:

Problema: El "Efecto Magnético" de la sección de tecnologías requiere calcular la posición del mouse constantemente. El uso de eventos tradicionales como mousemove puede saturar el procesador del navegador si se ejecutan demasiados cálculos por segundo, generando una navegación lenta o "lag".


Solución: Se utilizó la técnica de requestAnimationFrame para sincronizar los cálculos matemáticos de las tecnologías con la tasa de refresco del monitor. Esto optimizó el rendimiento, logrando que el movimiento de los iconos sea fluido y no afecte la velocidad de carga ni el desplazamiento (scroll) del resto del sitio

Cambios Durante el Desarrollo

De Estructura Rígida a Flexible

Originalmente se pensó en una plantilla idéntica para todos, pero se cambió hacia un modelo donde cada integrante gestiona su propia sección de manera autónoma, permitiendo mayor libertad creativa en la organización de la información.


Evolución del encabezado

Inicialmente el encabezado era estático, pero se decidió implementar una función de JavaScript que permite cambiar el color del título al hacer clic y alternar frases en distintos idiomas para darle un carácter global al proyecto.


Transición de Estilos en Tarjetas de Miembros

Originalmente las tarjetas eran estáticas, pero se decidió implementar efectos de transformación 3D (translateY y scale) junto con sombras dinámicas para mejorar la respuesta visual ante la interacción del usuario.


Implementación de Efectos Magnéticos en Tecnologías

Se añadió una función avanzada en JavaScript para la sección de tecnologías, permitiendo que los íconos (HTML, CSS, JS, etc.) sigan sutilmente el movimiento del mouse del usuario. Este cambio se realizó para añadir una capa de profundidad y modernidad al sitio que no estaba contemplada en el diseño inicial.
